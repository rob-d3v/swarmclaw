# SwarmClaw v1.6.2 Codex Continuity Release Plan

## Theme

v1.6.2 is a reliability follow-up for operators using CLI-backed agents, especially Codex. The release turns the PR #62 session-selection work into a safer patch release by making Codex resume state durable, clarifying active-session selection, and tightening CLI service behavior.

## Highlights

- **Codex sessions resume reliably.** Codex runs now use a stable per-session `CODEX_HOME` for every turn, so a thread emitted on a plain first turn has local metadata available on the next turn.
- **Resume IDs can be cleared intentionally.** Provider-mutated run-session resume fields now treat `null` as an intentional reset instead of falling back to stale persisted IDs.
- **CLI startup paths match user services.** The CLI can read `SWARMCLAW_HOME` from the user systemd unit, use `systemctl --user` for lifecycle commands when available, and find access keys under `$SWARMCLAW_HOME/platform-api-key.txt`.
- **Session focus is less surprising.** Active-session derivation prefers recent sessions with real content over empty thread sessions, and the inspector/chat surfaces show the selected session clearly.
- **Release risk is contained.** The unrelated lockfile metadata drift from PR #62 is excluded from the final merge.

## Verification Gate

Before tagging `v1.6.2`, run:

- `npm test`
- `npm run type-check`
- `npm run build:ci`
- `npm run test:e2e`
- `npm run electron:build` on the maintainer OS

Targeted regression checks already run during merge preparation:

- `node --import tsx --test src/lib/providers/cli-utils.test.ts`
- `node --import tsx --test --test-name-pattern 'codex' src/lib/server/chat-execution/chat-execution-session-sync.test.ts`
- `git diff --check github/main..HEAD`

## Release Steps

1. Confirm CI passes on the merge commit.
2. Confirm `npm view @swarmclawai/swarmclaw@1.6.2 version` is empty.
3. Confirm `git ls-remote --tags origin v1.6.2` is empty.
4. Bump package metadata to `1.6.2` only when ready to tag.
5. Tag `v1.6.2` and let the existing release workflows publish npm, GHCR, GitHub Release notes, and desktop artifacts.
